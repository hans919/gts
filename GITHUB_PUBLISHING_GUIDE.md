## âš ï¸ Problem: Can't Push to GitHub (10k File Limit)

GitHub Desktop has a limit of **10,000 files** per commit. The `node_modules` folder typically contains **20,000-50,000 files**, which is why you're getting this error.

---

## âœ… Solution: Exclude node_modules from Git

### Why You Should NOT Commit node_modules:

1. **Too Many Files** - Contains 20k-50k+ files
2. **Too Large** - Can be 100MB-500MB+ in size
3. **Platform Specific** - Some packages are OS-specific
4. **Unnecessary** - Can be regenerated with `npm install`
5. **Security** - May contain vulnerabilities that get patched

### âœ¨ The Correct Way:

**Use `.gitignore`** to exclude `node_modules` and let others install dependencies themselves.

### Step 3: Remove Tracked Files from Git

If `node_modules` was already committed to Git, remove it from tracking:

```powershell
# Navigate to project root
cd c:\xampp\htdocs\gts

# Remove node_modules from Git tracking (but keep the file)
git rm -r --cached frontend/node_modules
git rm -r --cached laravel/vendor
git rm -r --cached laravel/node_modules

# If you get "did not match any files" - that's okay, it means it wasn't tracked
```

---

### Step 4: Commit the Changes

```powershell
# Add all changes (respecting .gitignore)

# Commit

```

### Step 5: Push to GitHub

Now you should be able to push without the 10k file limit:

```powershell

```

Or use **GitHub Desktop**:

3. Add a commit message
4. Click "Commit to main"
